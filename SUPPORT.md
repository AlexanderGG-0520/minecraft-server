# Support

## Bugs

Use GitHub Issues for reproducible bugs.

Include:

* Image tag or commit.
* `TYPE` and `VERSION`.
* Relevant environment variables with secrets redacted.
* Kubernetes, Docker Compose, or Docker context.
* Logs around install, sync, shutdown, or runtime launch.
* Expected behavior and actual behavior.
* Any relevant manifests or minimal reproduction steps.

## Questions and design discussion

Use GitHub Discussions for design questions when available. If Discussions are not enabled, open an
issue and label it clearly as a question or design discussion.

## Security reports

Do not report security-sensitive issues in a public issue. Follow the guidance in
[SECURITY.md](SECURITY.md).

## Third-party mods and plugins

This project does not provide general support for arbitrary third-party mod, plugin, or modpack bugs.
Reports are in scope when they involve this image's lifecycle, install, sync, filesystem, shutdown, or
runtime behavior.
